# 사용자 친화적 UI 개선 포인트

> 작성일: 2026-04-11
> 대상 사용자: 백화점 지점 직원(비개발자), 한약국 약사, 본사 운영자
> 원칙: **"배워야 쓰는 시스템이 아니라, 보면 바로 이해하는 시스템"** (요구 문서 3-4)

---

## 🎯 핵심 원칙

1. **클릭 3번 안에 원하는 정보** — 자주 쓰는 작업은 대시보드나 헤더에서 1클릭
2. **색상과 아이콘으로 의미 전달** — 숫자/상태는 읽지 않고 눈으로 판단
3. **되돌릴 수 있음을 보장** — 삭제/수정은 확인 → 복구 안내
4. **검색창은 항상 포커스 가능** — 키보드 `/` 단축키로 어디서나 검색
5. **모바일에서도 동일 흐름** — 백화점 현장은 태블릿/폰 사용 비중 큼

## 🔴 우선순위 1 — 당장 체감되는 개선

### 1-1. 전역 검색바 (⌘K / Ctrl+K 또는 `/`)
- **문제**: 고객/제품/주문을 찾으려면 각 메뉴로 이동해야 함
- **개선**: 헤더 중앙에 검색바 — 고객명, 제품명, 주문번호, 송장번호를 한 번에 찾아 해당 페이지로 이동
- **효과**: 백화점 현장 "○○고객님 구매이력 확인" 요청 시 즉시 대응

### 1-2. POS — "자주 찾는 제품" 섹션
- **문제**: 제품 검색에 이름을 정확히 타이핑해야 함
- **개선**:
  - POS 화면 상단에 "자주 판매 Top 10" 버튼 그리드 (지점별 자동 학습)
  - 바코드 스캐너 포커스 자동 유지 (다른 입력창 포커스 시에도 스캐너 이벤트는 수신)
  - 키패드 수량 입력 (모바일 숫자패드)
- **효과**: 백화점 현장에서 제품 검색 시간 80% 절감

### 1-3. 대시보드 — "오늘의 할 일" 카드
- **문제**: 매출 숫자는 보이지만 "뭘 해야 하는지" 안내 없음
- **개선**: 대시보드 최상단에 Action 카드 섹션
  - 🔴 재고 부족 5개 품목 → 클릭 시 매입 화면으로
  - 🟡 송장번호 미등록 주문 12건 → 배송 화면으로
  - 🟢 발송 캠페인 대기 2건 → 캠페인 탭으로
  - 🔵 외상 미정산 8건 → 외상 탭으로
- **효과**: 아침 출근 후 "오늘 뭐부터 하지?" 해소

### 1-4. 빈 상태(Empty State) 개선
- **문제**: 데이터 0건일 때 빈 테이블만 표시 → 사용자 당황
- **개선**: 각 화면마다 Empty State 컴포넌트
  - "아직 고객이 없습니다. **+ 첫 고객 등록**" 버튼
  - "매입 이력이 없습니다. **매입 작성 가이드 보기**"
- **효과**: 신규 지점 온보딩 시 무엇을 해야 할지 자연스럽게 안내

### 1-5. 숫자 표시 통일
- **문제**: 천단위 콤마/원 접미 표기가 화면마다 다름
- **개선**: `<Money>` 컴포넌트로 전역 통일 (예: `1,234,567원`), 음수는 빨강
- **효과**: 숫자를 읽지 않고 스캔 가능

## 🟡 우선순위 2 — 사용 경험 향상

### 2-1. 액션 피드백 토스트
- **문제**: 저장/삭제 후 피드백이 alert 또는 없음
- **개선**: 우측 하단 토스트 — "저장되었습니다 · 되돌리기" 링크
- **효과**: 실수 삭제를 즉시 복구 가능

### 2-2. 필터/정렬 상태 URL 저장
- **문제**: 배송 목록에서 필터 걸고 상세 들어갔다 오면 필터 초기화
- **개선**: URL 쿼리스트링에 필터 상태 저장 — 새로고침/뒤로가기 해도 유지
- **효과**: 여러 건 순차 처리 시 흐름 유지

### 2-3. 모달 대신 Drawer (사이드 패널)
- **문제**: 수정 모달이 전체 화면을 가려 원본 데이터 대조 불가
- **개선**: 제품 수정, 고객 수정, 배송 수정은 우측 Drawer 패널로 — 뒤쪽 목록 그대로 보임
- **효과**: "이 고객 전화번호가 맞나?" 등 대조 작업 편리

### 2-4. 테이블 컬럼 사용자 지정
- **문제**: 테이블 컬럼이 많아 가로 스크롤
- **개선**: 톱니바퀴 아이콘 → 컬럼 on/off, 로컬스토리지 저장
- **효과**: 개인별 업무 스타일에 맞춤

### 2-5. 자주 쓰는 액션 단축키
- **개선**:
  - POS: `F1` 결제, `F2` 환불, `F3` 고객검색
  - 전역: `/` 검색, `G P` 제품페이지, `G C` 고객페이지 (gmail 스타일)
- **효과**: 숙련 사용자 속도 2배

### 2-6. 날짜 프리셋 버튼
- **문제**: 보고서/배송 목록에서 날짜 범위 설정이 번거로움
- **개선**: "오늘 / 어제 / 최근 7일 / 이번달 / 지난달" 버튼 고정
- **효과**: 대부분의 케이스를 1클릭으로
- **현황**: 배송 목록에는 "최근 1주" 버튼 방금 추가됨 → 다른 화면에도 일관 적용

### 2-7. 고객 상세 — 한 화면에 모두
- **문제**: 고객 상세에서 구매이력/상담/포인트/등급이 탭으로 분리되어 전환 필요
- **개선**: 상단 요약 카드(등급/누적구매/최근방문/포인트) + 하단 세로 스크롤 섹션(구매→상담→포인트 순)
- **효과**: 약사가 고객 응대 시 스크롤만으로 모든 정보 확인

### 2-8. 입력 검증 인라인 표시
- **문제**: 제출 후 에러 메시지 → 다시 입력
- **개선**: 입력 즉시 검증 (전화번호 형식, 이메일, 음수 금액), 녹색 체크/빨강 X
- **효과**: 제출 실패 경험 제거

## 🟢 우선순위 3 — 장기 개선

### 3-1. 온보딩 투어 (Driver.js 스타일)
- 첫 로그인 시 역할별 5단계 가이드 팝업
- 주요 기능 위치를 하이라이트로 안내
- 건너뛰기/다시보기 가능 (`users.onboarding_completed` 플래그)

### 3-2. 컨텍스트 도움말 (? 아이콘)
- 각 화면 우상단 `?` 아이콘 → 해당 화면 사용법 사이드 팝업
- 마크다운 기반으로 비개발자도 편집 가능

### 3-3. 다크모드 (지점 밝은 조명용)
- 백화점 매장 조명이 강해 화면 눈부심
- 시스템 설정 연동 또는 수동 토글

### 3-4. 알림 센터 (벨 아이콘)
- 헤더에 벨 아이콘 → 본인 관련 알림 목록
  - 담당 고객 생일 D-3
  - 재고 부족 품목 신규 발생
  - 본사 공지사항
- 안 읽은 카운트 뱃지

### 3-5. 프린터 프리셋
- POS 영수증: 지점별 프린터 설정 저장 (58mm/80mm/A4)
- 배송 송장 라벨: 용지 규격 기본값 저장

### 3-6. 사용자 피드백 버튼
- 좌하단 "피드백 보내기" 버튼 → 현재 화면 스크린샷 + 텍스트 → Slack/이메일 전송
- 현장 실사용자 목소리 즉시 수집

## 🎨 디자인 시스템 정리 권장

현재 개별 화면에 하드코딩된 스타일이 많습니다. 다음을 컴포넌트화하면 일관성과 유지보수성이 오릅니다:

```tsx
<PageHeader title="배송 관리" subtitle="카페24 + 직접입력">
  <PrimaryAction>+ 새 배송</PrimaryAction>
</PageHeader>

<FilterBar>
  <DateRangePreset defaultValue="last7" />
  <StatusFilter options={STATUS_OPTIONS} />
  <SearchInput placeholder="수령자/전화/송장" />
</FilterBar>

<DataTable
  columns={columns}
  rows={filteredShipments}
  onRowClick={openDetail}
  bulkActions={bulkActions}
  emptyState={<EmptyState message="배송 내역이 없습니다" />}
/>

<Drawer open={editing}>
  <ShipmentForm ... />
</Drawer>

<Toast />
```

컴포넌트 5~10개만 정리해도 전체 화면이 통일됩니다.

## ✅ 구현 난이도 매트릭스

| 개선 | 난이도 | 체감 효과 | 권장 순서 |
|---|---|---|---|
| 빈 상태 개선 | 낮음 | 중 | 1 |
| 날짜 프리셋 통일 | 낮음 | 높음 | 2 |
| 숫자 Money 컴포넌트 | 낮음 | 중 | 3 |
| 토스트 피드백 | 낮음 | 중 | 4 |
| 오늘의 할 일 카드 | 중 | 매우 높음 | 5 |
| POS 자주 찾는 제품 | 중 | 매우 높음 | 6 |
| 전역 검색 ⌘K | 중 | 매우 높음 | 7 |
| Drawer 패널 | 중 | 중 | 8 |
| 컴포넌트 시스템 정리 | 높음 | 높음(장기) | 9 |
| 온보딩 투어 | 높음 | 중 | 10 |

**즉시 착수 권장**: 1~4 (저난이도·즉효). 이후 5~7은 본사/지점 실사용자 피드백을 받아 우선순위 조정.
