## 2. 근본 원인

레거시 적재 시 `legacy_orders.branch_id`를 **매출처 이름(`channel_text`) 정확일치**로만 매핑 →
이름 변형·신규 추가된 백화점/팝업/온라인몰 매출처가 연결되지 않음.
미매칭 24.26억은 **전액 아래 매출처들**임(일반 매장 누락 없음).

## 4. 결정 필요 항목

1. **강남신세계(팝업) 4.65억** — 별도 EVENT 지점으로 둘지, 정식 강남신세계에 통합할지
   (팝업 23-09~25-08 → 정식 25-08~, 같은 위치의 연속). 기존 타 팝업들은 별도 EVENT로 유지 중.
2. **신규 EVENT 지점 생성** — 명동롯데/하이산홍콩/한남나인원 팝업 (각각 생성 vs 유사채널 통합).
3. **폐점·종료 매출처 표시** — 현재 종료 팝업은 `is_active=false` EVENT. 매출 조회엔 포함(과거 누적 보존),
   POS 매출처/출고처 선택에선 숨김(현 동작). 이 기준 확정.
4. **비매출처 코드** — '원장님'(0.08억)·'매니저(대신)'(0원)을 본사 귀속/기타 처리/제외 중 선택.

## 5. 적용 방법(결정 확정 후)

- `legacy_orders.branch_id` **백필 UPDATE**: 이름 정규화(공백 제거)+별칭맵으로 `channel_text` → `branches.id` 연결.
  레거시 적재와 동일 방식(휴리스틱 아님, 명시 매핑표). 원천 `channel_text`/`branch_code_raw`는 무손상.
- 필요 시 누락 팝업 EVENT 지점 신규 생성 후 연결.
- 검증: 백필 후 미매칭 금액 0(또는 의도적 '기타'만) 확인 → `branch_sales_summary` 재조회로 지점별 정합 확인.

> ⚠️ 운영 데이터 UPDATE이므로 **결정 사인오프 후** 실행. 본 문서는 진단·제안 단계.
