## 1. 대시보드 `/`

### ① 기본
- [ ] 오늘 매출 카드 숫자 = `sales_orders` 오늘자 합계 + B2B 납품 오늘자 합계
- [ ] 이번달 매출 카드 = 당월 누적
- [ ] 이번달 환불 카드 + 순매출 = 매출 - 환불
- [ ] 채널별 매출 (매장/백화점/온라인/행사) 표시
- [ ] 지점별 재고 상태 카드 — 부족/정상 뱃지 정확
- [ ] 최근 주문 6건 표시 (Cafe24 온라인 뱃지 포함)

### ② 권한
- [ ] `BR`/`PH` 로그인 시 본인 지점 데이터만 노출 (HQ 뷰 전환 불가)
- [ ] `HQ`/`SA`는 '전체' ↔ '지점 뷰' 전환 버튼 동작
- [ ] `EX` 로그인 시 대시보드 접근 가능 / 편집 기능 없음

### ③ 엣지
- [ ] 데이터 0건 상태 (신규 지점) — 카드 0원 표시, 에러 없음
- [ ] 주문 1건만 있을 때 — 채널별 매출 그래프 정상
- [ ] 월 첫째날 자정 경계 — 어제 월말 주문이 이번달에 끼지 않음

### ④ 연계
- [ ] POS에서 결제 완료 → 대시보드 오늘 매출에 즉시 반영 (새로고침 후)
- [ ] B2B 납품 등록 → 오늘/월간 매출에 합산

## 2. POS `/pos`

### ① 기본
- [ ] 제품 검색 → 장바구니 담기
- [ ] 수량 조정 / 삭제 / 장바구니 초기화
- [ ] 고객 검색/선택 (전화번호·이름)
- [ ] 할인 적용 (금액/%) 후 합계 재계산
- [ ] 결제수단 선택: 카드(단말기)/현금/계좌이체/외상/포인트 혼합
- [ ] 영수증 모달 출력 → 인쇄 버튼 동작
- [ ] 환불 모달 — 기존 주문 불러오기 → 환불 처리

### ② 권한
- [ ] `BR`/`PH` 로그인 시 본인 지점만 POS 사용
- [ ] `HQ`/`SA`는 지점 선택기 노출

### ③ 엣지
- [ ] 재고 0인 제품 담기 시도 → 경고
- [ ] 고객 미선택 + 결제수단 '외상' → 차단 (외상은 고객 필수)
- [ ] VAT 계산 검증: 공급가 = price ÷ 1.1, VAT = price × 10/110
- [ ] 포인트 잔액 초과 차감 시도 → 차단
- [ ] 네트워크 끊김 중 결제 버튼 연타 → 중복 결제 방지

### ④ 연계
- [ ] 결제 완료 → `inventories` 차감, `inventory_movements` OUT 기록
- [ ] 결제 완료 → `point_history` 적립 (등급 자동 재계산)
- [ ] 결제 완료 → 회계 분개 자동 생성 (`journal_entries`)
- [ ] 외상 결제 → 거래관리/외상 탭에 노출, `credit_settled=false`
- [ ] NICE 단말기 브릿지 연동 (준비 중) — 승인 완료 후 결제 확정 2단계 동작

### 🏬 백화점 모드
- [ ] 지점이 `channel=DEPT_STORE`인 경우 백화점 모드 자동 전환
- [ ] 카드 결제 간소화 UI (단말기 승인번호 수기 입력)
- [ ] 백화점 카드 결제 시 `payment_method='card'` + 승인번호 저장
